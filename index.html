How It Works: The Logic Flow

To achieve lifelike movement on a microcontroller, this system employs a smart buffer manipulation strategy:

  • Boot Load: BMP assets (iris, pupil, eyelid) are written directly to the display's internal controller memory at startup.
  • The Window Trick: Instead of sending 57,600 pixels every frame, the code updates only the specific coordinates where the pupil or eyelid is moving.
  • Randomized Expression: Built-in "look and blink" algorithms simulate natural behavior, preventing a static "uncanny" look.
  • Interaction: Ready to accept external $X, Y$ coordinates via UART for active human face tracking.

Technical Logic & Buffer Management

The system overcomes the RP2040 SPI bus bottlenecks by using two critical optimization techniques:

  • DMA (Direct Memory Access): Frees the CPU to calculate pupil math while the DMA controller handles high-speed data transmission to the displays.
  • Corner Masking: Since the GC9A01 is round but driven by a $240 \times 240$ controller, the code skips calculating "corner" pixels, reducing redundant processing by ~21%.
