How It Works: The Logic Flow

The system is designed to provide maximum visual fluidity with minimal computational cost:

  • Boot Load: BMP assets (iris, pupil, eyelid) are loaded into the display's internal memory at startup.
  • Partial Refresh: The RP2040 tracks coordinates and updates only the moving parts of the eye.
  • Blink Logic: An asynchronous timer triggers natural-looking blinks that don't interrupt tracking.
  • Tracking Integration: Receives external $X, Y$ data from the FaceTracker via UART for active interaction.

Synchronized Movement

The system drives two independent round LCDs simultaneously, ensuring that blinking and pupil movement are perfectly synchronized for a cohesive facial expression.

Active Face Tracking

Designed to maintain "eye contact." By pairing this with an ESP32-CAM, the robot follows human movement in real-time across a room.
